并查集 (Union Find) 算法

这几天重温一下union-find算法, 重温 《算法》(Algorithms) 这本书union find 一小节, 写一篇学习笔记, 然后结合一些实际的算法例子的一篇读后感.

Union-fina 算法主要是用来解决动态连通性问题的. 那么什么是动态连通性问题, union-find又是如何解决这类问题的, 下面将展开一些细节

动态连通性问题 (Dynamic connectivity Problem)

给定一个pair(p, q), 如果p和q尚未连通,则把二者连通起来. p和q连通后, 称p和q在同一组内.

动态连通性的应用场景：

网络连接判断：

如果每个pair中的两个整数分别代表一个网络节点，那么该pair就是用来表示这两个节点是需要连通的。那么为所有的pairs建立了动态连通图后，就能够尽可能少的减少布线的需要，因为已经连通的两个节点会被直接忽略掉。

变量名等同性(类似于指针的概念)：

在程序中，可以声明多个引用来指向同一对象，这个时候就可以通过为程序中声明的引用和实际对象建立动态连通图来判断哪些引用实际上是指向同一对象。

例如:

简言之, 即给定两点(p, q), 如果没有连接, 那么根据已有的连通性判断给定两点是否连接,如果没有连接,那么就连接起来, 如果已经连接, 那么就不需要重新连接. 如图, A (0, 1, 2, 5 , 6) 和 B (3, 4, 7) 已连通, A中和B中任何两点都是不连通的, 如果给定两点(2,4)是不连通, 那么union-find 会把两点连通, A 和 B 中任意两点就都连通了.

图中 A 和 B 分别是两个连通的 component, 我们把连通看做一种等价, 连通的component看作是等价类, 图中 A 和 B 分别为两个等价类 (即连通分量).

连通特性

假设p和q已经相连(connected), 那么 - 自反性(Reflexive): p 和 q 是相连的. - 对称性(Symmetric): 如果 p 和 q 是相连的, 那么反过来 q 和 p 也是相连的 - 传递性(Transitive): 如果 p 和 q 相连, 且 p 和 r 也相连, 那么 q 和 r 也是相连的. 图中例子, 0 和 1 相连, 0 和 5 相连, 那么 1 和 5 也是相连的.

在一个图中, 如何快速的找到两点是否连通, 如何快速的连通两点? Union-find可以很好解决这个问题.

Union-Find

实现: 利用数组实现, 数组下标表示没个数.

API: - int find(int p): 找出给定点p的等价类(连通分量). - void union(int p, int q): 给两个点建立连接, 使之构成一个等价类(连通分量). - boolean connected(int p, int q): 判定两点p 和 q 是否在同一等价类(连通分量)中. - int count(): 一个图中有多少个等价类(连通分量).

Note: 这里用整数表示节点, 如果是其他类型, 则可以用HashMap来建立相应的映射.

Quick-Find 算法:

数据结构: 1. Integer array id[] of size N.

int[] id = new int[N];
  1. 如果p和q是连通的, 那么p和q会有相同的id.

例如:

Java 代码实现

public class QuickFind {
    int[] id;
    public QuickFind(int N) {
        id = new int[N];
        for (int i = 0; i < N; i++) {
            id[i] = i;
        }
    }

    public boolean find(int p, int q) {
        return id[p] == id[q];
    }

    public void union(int p, int q) {
        int pid = id[p];
        int qid = id[q];
        for (int i = 0; i < id.length; i++) {
            if (id[i] == qid) {
                id[q] = pid;
            }
        }
    }
}

####时间复杂度: - find(): 时间复杂度是O(1) - union(): 时间复杂度是O(N)

显而易见, Quick-Find 算法太慢了, Quick-Find 算法, 可能要·O(MN)计算. 如果新添加路径的数量是M，节点数量是N，那么最后的时间复杂度就是MN，显然是一个平方阶的复杂度，对于大规模的数据而言，这是非常非常慢的(给个如下图数据), 想要解决这个问题，关键就是要提高union方法的效率，让它不再需要遍历整个数组.

Quick-Union 算法

数据结构: 1. Integer array parent[] of size N 2. parent[i] 是i节点的父节点 3. Root 节点 i 是 parent[parent[...preant[i]...]]

例子:

要连通pair (3, 5), 找到3和5的root, 只要把5的root 变成3的root即可. 如下图所示:

Java 代码实现:

public class QuickUnion {
    int[] parent;
    public QuickUnion(int N) {
        parent = new int[N];
        for (int i = 0; i < N; i++) {
            parent[i] = i;
        }
    }

    public int root(int p) {
        while (p != parent[p]) p = parent[p];
        return p;
    }

    public boolean find(int p, int q) {
        return root(p) == root(q);
    }

    public void union(int p, int q) {
        int rp = root(p);
        int rq = root(q);
        parent[rp] = rq;
    }
}

时间复杂度:

  • find() 最坏的情况下时间复杂度是 O(N), 当树不平衡, 每个root 只有一个子节点的时候. 如:

  • union() 时间复杂度是 O(1), 如果包含 find() 的时间复杂度, 那么最坏情况下时间复杂度也是 O(N)

Weighted quick-union 算法 (加权quick-union)

Quick-Union 查找慢是由于当树变的很高的时候, 那么要加快就可以平衡树使得树不变高.

数据结构: - Integer array parent[] of size N - Integer array rank[] of size N - parent[i] 是 i 节点的 root 节点 - rank[i] 是 i 节点所在component 的个数

Java算法实现

public class WeightedQuickUnion {
    int[] parent;
    int[] rank;
    public WeighterQuickUnion(int N) {
        parent = new int[N];
        rank = new int[N];
        for (int i = 0; i < N; i++) {
            parent[i] = i;
            rank[i] = 1;
        }
    }

    public int root(int x) {
        return parent[x] == x ? x : root(parent[x]);
    }
    
    public boolean find(int p, int q) {
        return root(p) == root(q);
    }
    
    public void union(int p, int q) {
        int rp = root(p);
        int rq = root(q);
        if (rank[rp] >= rank[rq]) {
            parent[rq] = rp;
            rank[rp] += rank[rq];
        } else {
            parent[rp] = rq;
            rank[rq] += rank[rp];
        }
    }
}

时间复杂度

  • find() 时间复杂度 log(N)
  • union() 时间复杂度 log(N)

*Quick-Union*和Weighted-Quick-Union 的比较:

通过比较, 我们可以发现,用rank去记录每个连通component的size, 然后平衡树, 即使得树扁平化(如上图), 可以避免树变得很高, 提高树查找的时间复杂度.

以下是几种union-find 算法的时间复杂度的比较:

根据上图对不同union-find算法的时间复杂度的分析, 对于大规模数据的运算, 优先选择Weighted Quick-Union 算法. 如果数据量超大, 那么最好选 Weighted Quick-Union with Path Compression 算法.

Leetcode: 323. Number of Connected Components in an Undirected Graph

Given n nodes labeled from 0 to n - 1 and a list of undirected edges (each edge is a pair of nodes), write a function to find the number of connected components in an undirected graph.

Example 1:

Input: n = 5 and edges = [[0, 1], [1, 2], [3, 4]]

     0          3
     |          |
     1 --- 2    4 

Output: 2
Example 2:

Input: n = 5 and edges = [[0, 1], [1, 2], [2, 3], [3, 4]]

     0           4
     |           |
     1 --- 2 --- 3

Output:  1
Note:
You can assume that no duplicate edges will appear in edges. Since all edges are undirected, [0, 1] is the same as [1, 0] and thus will not appear together in edges.

这是一道非常经典用Union-Find解答的题, 题目大意是给定一些节点, 和一些相连的pair 对(连通pair), 求这些点中, 有多少个连通图.

分析: 这是一个图, 要求有多少个连通图, 用Union-Find, 对每个pair对, 查看这个pair 对是否已经相连, 若没有相连, 则Union. 最后找出总共有多少个连通图.

Java代码实现:

class Solution {
    public int countComponents(int n, int[][] edges) {
        if (edges == null || edges.length == 0) return n;
        UnionFind uf = new UnionFind(n);
        for (int[] edge : edges) {
            int px = uf.find(edge[0]);
            int py = uf.find(edge[1]);
            if (px == py) continue;
            uf.union(edge[0], edge[1]);
        }
        return uf.numConnected;
    }
}
class UnionFind {
    int[] parent;
    int[] rank;
    int numConnected;
    public UnionFind(int k) {
        parent = new int[k];
        rank = new int[k];
        numConnected = k;
        // pre fill parent and rank
        for (int i = 0; i < k; i++) {
            parent[i] = i;
            rank[i] = 1;
        }
    }

    public int find(int x) {
        int p = parent[x];
        if (p == x) return x;
        return parent[x] = find(parent[x]);
    }
    
    public void union(int x, int y) {
        int px = find(x);
        int py = find(y);
        if (px == py) return;
        if (rank[py] >= rank[px]) {
            parent[px] = py;
            rank[py] += rank[px];
        } else {
            parent[py] = px;
            rank[px] += rank[py];
        }
        numConnected--;
    }
}
