创建和发布第一个Hugo site 到你的 Github pages

Posted by     "Snowan" on Monday, June 3, 2019

这篇文章将要带你一步一步创建第一个Hugo静态网页, 并发布到Github pages.

安装 Hugo

快速安装 hugo

Homebrew 安装(macOS)

brew install hugo

更多其他OS安装请查看 Hugo installing

查看时候安装成功: hugo version

> hugo version 
Hugo Static Site Generator v0.55.6/extended darwin/amd64 BuildDate: unknown

如果看到Hugo 版本号信息, 那么恭喜你已经成功安装 Hugo.

创建 Hugo site

打开 Terminal, 输入命令行创建第一个hugo site (first_hugo_site), 并进入到site当前目录.

> hugo new site first_hugo_site
> cd first_hugo_site

到这里, 你已经成功创建了一个空的hugo 静态网页, 对, 就是这么简单… 现在我们可以加一些主题, 内容, 并让你的网页跑起来.

添加主题(theme)

你可以到Hugo主题栏, 有很多简洁优美的主题, 挑选一个适合你的吧. Hugo主题在这里找到. 我个人Github Pages用的是hugo-theme-cleanwhite主题. 接下来的例子中我都会用这个主题来介绍.

  1. 创建一个Github repo (first_hugo_site), 如果你还没有Github 账号, 可以在这里申请一个. 现在我们有了 first_hugo_site repo, 在当前目录下初始化git

    first_hugo_site > git init

  2. 添加Hugo 主题到你的site, 首先创建一个名为themes文件夹, 然后用 git submodule add ... 添加主题,

    first_hugo_site > mkdir themes
first_hugo_site > git submodule add https://github.com/zhaohuabing/hugo-theme-cleanwhite.git themes/hugo-theme-cleanwhite

  3. 在本地 run hugo server,

    first_hugo_site > hugo serve -t hugo-theme-cleanwhite

    现在你应该在localhost:1313能看到hugo主题已经up, 如下所示, 然后你可以修改配置(config.toml), customize成你自己的blog页面. 具体如何修改,请参考这里.

Deploy

到这里, 我们已经创建好了基本的hugo site, 接下来要deploy hugo site 到Github pages, 这里需要你创建Git repo 来host GitHub pages. 所以你需要2个 Github Repos, 一个repo是 Hugo site, 另一个是用来hold给 github pages 生成的HTML, 进行加载和渲染, 内容可以真正在你的Github pages 上显示. 两个Git repos: first_hugo_site, <username>.github.io (这个是你的 github pages URL).

新建Git repo, 可以参考这里.

现在我们已经有了两个repos, 接下来我们需要把hugo site 和 github pages 连接起来.

// 添加remote repo
first_hugo_site > git remote add origin https://github.com/<githubusername>/first_hugo_site.git

// 添加github pages repo as submodule
first_hugo_site > git submodule add https://github.com/<githubusername>/<githubusername>.github.io.git

// 添加本地所有改动
first_hugo_site > git add . 

// commit
first_hugo_site > git commit -m "First commit first hugo site"

// push 本地文件到远程git repo
first_hugo_site > git push origin master

到目前为止, 我们已经成功push 本地改动到远程的 first_hugo_site repo. 我们还需要生成HTML 页面给GitHub Pages 渲染和加载. hugo 会生成文件到public文件夹, 现在我们要修改config.toml, 使得新生成的HTML文件能够发布到submodule <githubusername>.github.io 网站.

first_hugo_site > vim config.toml

  baseurl = "https://<githubusername>.github.io"
  title = "Your Blog"
  theme = "hugo-theme-cleanwhite"
  languageCode = "en-us"
  # Enable comments by entering your Disqus shortname
  paginate = 5 #frontpage pagination
  hasCJKLanguage = true
  publishDir = "<githubusername>.github.io"
...

  • baseurl 设置你的blog url, 所有生成的HTML 和内容就会发布到url

  • publishDir 告诉Hugo 所有新生成的HTML 和内容都要发布到目录下 (在 first_hugo_site 有submodule <githubusername>.github.io)

用 hugo 生成HTML和内容到 <githubusername>.github.io submodule,

first_hugo_site > hugo

到这里, 已经一切准备就绪, 现在submodule<githubusername>.github.io 已经有生成的HTML, CSS 等内容, 最后一步就是push 到远程的 <githubusername>.github.io repo, 然后你的GitHub pages 就可以上线了.

// 到submodule 目录下
first_hugo_site > cd <githubusername>.github.io

// 添加所有本地改动
first_hugo_site/<githubusername>.github.io > git add .

// commit 所有添加的本地修改
first_hugo_site/<githubusername>.github.io > git commit -m "generate html and content"

// push 到 远程repo
first_hugo_site/<githubusername>.github.io > git push origin master

等一小会(心急吃不热热豆腐, 😁), 去到你的github page 页面, https://<githubusername>.github.io, 现在你应该能看到你的第一个Hugo site (with theme) on Github pages. 例子 如下:

到这里你已经成功创建和发布了第一个Hugo site 到Github pages. 接下来让我们一起发布内容到blog.

创建content

用hugo, 你可以创建一些内容, 用hugo 命令行

// 会在content文件夹下创建一个 posts 文件夹, ../content/posts/my-first-post.md
first_hugo_site > hugo new posts/my-first-post.md

// content/posts/content/articles/my-first/article.mds/my-first/article.md
first_hugo_site > hugo new articles/my-first-article.md

以hugo-theme-cleanwhite 主题为例, 写一个md例子

first_hugo_site > hugo new posts/hello-world.md
first_hugo_site > vim hello-world.md
  ---
  layout: post
  title:       "Snowan Hello World post"
  subtitle:    ""
  description: ""
  date:        2019-06-03
  author:      "Snowan"
  image:       ""
  ---

  ### Hello World!

  > "Yeah, Snowan blog is on!!"
