Leetcode 465 Optimal Account Balancing 解题报告

题目 (Problem)

A group of friends went on holiday and sometimes lent each other money. For example, Alice paid for Bill’s lunch for $10. Then later Chris gave Alice $5 for a taxi ride. We can model each transaction as a tuple (x, y, z) which means person x gave person y $z. Assuming Alice, Bill, and Chris are person 0, 1, and 2 respectively (0, 1, 2 are the person’s ID), the transactions can be represented as [[0, 1, 10], [2, 0, 5]].

Given a list of transactions between a group of people, return the minimum number of transactions required to settle the debt.

Note:

  1. A transaction will be given as a tuple (x, y, z). Note that x ≠ y and z > 0.

  2. Person’s IDs may not be linear, e.g. we could have the persons 0, 1, 2 or we could also have the persons 0, 2, 6.

    Example 1:

Input:
[[0,1,10], [2,0,5]]

Output:
2

Explanation:
Person #0 gave person #1 $10.
Person #2 gave person #0 $5.

Two transactions are needed. One way to settle the debt is person #1 pays person #0 and #2 $5 each.


Example 2:

Input:
[[0,1,10], [1,0,1], [1,2,5], [2,0,5]]

Output:
1

Explanation:
Person #0 gave person #1 $10.
Person #1 gave person #0 $1.
Person #1 gave person #2 $5.
Person #2 gave person #0 $5.

Therefore, person #1 only need to give person #0 $4, and all debt is settled.

题意:

题中意思是一堆朋友一起出去游玩,然后分享账单,一个人给另一个人一些钱吃饭,一个人给另外一个人车费,最后有一些账单,你可能欠一个人钱, 另一个人可能欠你的钱的账单, 最后问要怎么样最简单的能够把账单结清. (像一个分账单的app splitwise), 最后有个选项是smart group.

思路:

最后账单总和为0, 因为要找到最简单的方式把账单结清, 所以如果有人不欠别人的钱的话, 就没有必要继续计算此人了. 那么要怎么直到每个人有多少账单, 可以用一个HashMap, Person ID 作为 Key, 账单(钱)作为 Value. 扫描一遍给定的tuple, 给钱的加上钱, 欠钱的减去. 如, [0,1,5], person 0给person 1 $5.所以 person 0 +5, person 1 -5. 最后得到Map, person 和对应的debt, 对于debt 为0的person, 我们不需要做多余的分配, 因为这里求最少的分配方式. 由于不需要记录具体如何分配, 也就是谁给谁多少钱, 所以从Map中, 取出value不是0的值组成一个数组, 那么现在就变成怎么样, 如何分配, 够使得所有的值最后都是0的问题?

对于每一个有debt的person, 进行清零操作. 从第一个debt(index 0)开始, 往后找到index i 如果debts[i]跟debts[0]的符号相反, 进行相加, 这样可以平衡两者之间的负债情况, 把当前debts[i] = debts[0] + debts[i], 可以到达0的状态, 继续进行下一个,直到每一个debts都为0. 可以维护一个最小交易数的变量, 用递归清零操作, 最后找到最小. Skip已经清零的账户, 继续往后查找.

假设有: [5, -10, 5], 我们直到2次转账就可以达到平衡, person 1 分别给person 0 和 2 $5.

时间复杂度 (TC): O(n*2^n) - n is the size of debts

空间复杂度 (SC): O(n*2^n)

Java code:

public static int minTransfers(int[][] transactions) {
    if (transactions == null || transactions.length == 0) return 0;
    int len = transactions.length;
    if (len == 1) return 1;
    Map<Integer, Integer> balanceMap = new HashMap<>();
    for (int[] tc : transactions) {
      balanceMap.put(tc[0], balanceMap.getOrDefault(tc[0], 0) + tc[2]);
      balanceMap.put(tc[1], balanceMap.getOrDefault(tc[1], 0) - tc[2]);
    }
    List<Integer> debts = balanceMap.values().stream()
        .filter(v -> v != 0)
        .collect(Collectors.toList());
    return helper(0, debts);
  }

  private static int helper(int idx, List<Integer> debts) {
      // skip 0
    while (idx < debts.size() && debts.get(idx) == 0) idx++;
    if (idx == debts.size()) return 0;
    int counts = Integer.MAX_VALUE;
    for (int i = idx + 1; i < debts.size(); i++) {
      // can be balanced
      if (debts.get(i) * debts.get(idx) < 0) {
        debts.set(i, debts.get(i) + debts.get(idx));
        counts = Math.min(counts, 1 + helper(idx + 1, debts));
        debts.set(i, debts.get(i) - debts.get(idx));
      }
    }
    return counts;
  }
